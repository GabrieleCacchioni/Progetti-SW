Guida per configurare la GUI     (In fondo riporto gli aggiornamenti)

1. Bisogna creare un file dei parametri col nome 'nome_modello_conf.xml'
2. Il file deve essere strutturato con una radice <parameters> e poi dentro 
			-un figlio <frame>
			-un figlio <buttons>
			-un figlio <defaultb>

3.Il <frame> deve avere due figli siffatti
					<width> valore </width>
					<heigth> valore </heigth>
	non necessariamente in questo ordine
					
4.Il <buttons> ha n figli dove n è il numero di eventi riportati nell'ordine di apparizione nella state chart
  e strutturati nel seguente modo
  				<button id="nome_evento">
  					<color expr="nome_colore" />
  					<tooltip expr="il tooltip" />
					</button>
	I parametri possono mancare, ad esempio <color expr=""> va bene ma devono essere scritti entrambi anche se vuoti
	
5.Il <defaultb> rappresenta i valori di default per i buttons e deve avere un dcolor e un dtooltip come sopra.

ESEMPIO DI FILE PARAMETRI

<parameters>
	<frame>
		<width> n </width>
		<heigth> m </heigth>
	</frame>
	
	<buttons>
		<button id="evento_prova">
			<color expr="red" />
			<tooltip expr="evento di prova" />
		</button>
	</buttons>
	
	<defaultb>
		<dcolor expr="white" />
		<dtooltip expr="default" />
	</defaultb>

</parameters>

con n e m interi (sono i pixel!!!)

ATTENZIONE: Se una di queste cose non viee rispettata il programma non va!(per ora -> LEGGERE SOTTO)



################################
AGGIORNAMENTO 1
################################
Il programma gira anche nelle seguenti eventualità

1. Il frame non viene dichiarato, viene dichiarato con parti mancanti o valori mancanti
2. I buttons vengono dichiarati con parti mancanti o valori assenti, per ora però tutti i buttons devono essere ancora dichiarati

Quello che è invece consigliato dichiarare (a meno che non si dichiari il resto nei MINIMI dettagli) sono i default, ora oltre al defaultb è consigliato
dichiarare anche un defaultf come uno qui sotto:

	<defaultf>
		<width> 300 </width>
		<heigth> 300 </heigth>
	</defaultf>

AGGIORNAMENTO 3
###############################
Ora anche i pannelli che riprtano i nomi delle variabili sono parametrici! Vanno pertanto specificati un colore ed un tooltip. Anche qui è ESSENZIALE
indicare ALMENO il default, il resto si può evitare. Come nell'aggiornamento 2 BISOGNA dare, nel file dei parametri, lo stesso nome ai panel delle variabili
(si può anche non darlo ma nel caso decideste di parametrizzare un panel dovrete rispettare il nome).

NOTE:
i parametri di colore e tooltip per i Panel devono chiamarsi pcolor e ptooltip, come si vede anche sotto.
Per il default invece è come button, ossia basta dcolor e dtooltip.

Nel file dei parametri, vanno quindi aggiunti(dove vi pare e non in questo ordine, ma come figli diretti di parameters) [RICORDA solo il Default è obbligaotrio]

	<panels>
		<panel id = "nome_var">
			<pcolor  expr= "COLORE" />
			<ptooltip ecpr="tooltip che si vuole" />
		</panel>
	</panels>
	
	<defaultp>
		<dcolor expr="COLORE_DEFAULT" />
		<dtooltip expr="tooltip di default per panels" />
	</defaultp>
	
RICORDO anche che tutto questo è funzionante col model stopwatch e trovate già un file di configurazione che ho creato io nella cartella del progetto
